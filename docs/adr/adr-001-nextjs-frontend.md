# ADR-001: استخدام Next.js للتطبيق الأمامي

## الحالة
مقبول (Accepted)

## السياق
نحتاج إلى إطار عمل للواجهة الأمامية يدعم:
- تحسين محركات البحث (SEO)
- التصيير من جانب الخادم (SSR)
- التصيير الثابت (SSG)
- الأداء العالي
- تجربة مطور ممتازة

## الخيارات المدروسة

### الخيار أ: Next.js (App Router)
- إيجابيات: SSR/SSG مدمج، SEO ممتاز، App Router حديث، نظام توجيه قوي، Server Components، مجتمع كبير
- سلبيات: منحنى تعلم App Router، بعض التعقيد في النشر

### الخيار ب: React + Vite (SPA)
- إيجابيات: بسيط، سريع في التطوير، Vite سريع
- سلبيات: لا SSR، SEO ضعيف، يحتاج حلول إضافية للـ SEO

### الخيار ج: Remix
- إيجابيات: SSR ممتاز، بسيط
- سلبيات: مجتمع أصغر، نظام بيئي أقل نضجاً

## القرار
اخترنا **Next.js مع App Router** للأسباب التالية:
1. SEO أساسي للموقع الرسمي والمتجر
2. SSR يحسن أداء التحميل الأولي
3. Server Components يقلل JavaScript المرسل للمتصفح
4. نظام التوجيه (App Router) حديث وقوي
5. مجتمع كبير ودعم طويل الأمد من Vercel

## العواقب
- إيجابيات: SEO + SSR + أداء + تجربة مطور
- سلبيات: تعقيد إضافي مقارنة بـ SPA البسيطة
- مخاطر: الاعتماد على Vercel (يمكن تخفيفها بالنشر الذاتي)
