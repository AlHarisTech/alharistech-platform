# وثيقة متطلبات المنتج (PRD)
## Product Requirements Document — AlharisTech Platform v1.0

---

## 1. نظرة عامة على المنتج

منصة الحارس تك هي منصة رقمية مؤسسية متكاملة تخدم مؤسسة الحارس تك للخدمات. تهدف إلى رقمنة كافة عمليات المؤسسة وتقديم خدماتها عبر قنوات متعددة (Web, Desktop, Mobile, API).

---

## 2. الأهداف (Objectives)

| الهدف | الوصف | مقياس النجاح |
|:---|:---|:---|
| O1 | إنشاء وجود رقمي احترافي | إطلاق الموقع الرسمي |
| O2 | تحويل الخدمات اليدوية إلى رقمية | استقبال الطلبات إلكترونياً |
| O3 | إدارة العملاء مركزياً | لوحة تحكم موحدة |
| O4 | تمكين التجارة الإلكترونية | متجر إلكتروني متكامل |
| O5 | دعم القنوات المتعددة | تطبيقات Desktop + Mobile |
| O6 | تمكين الذكاء الاصطناعي | AI Assistant + Automation |

---

## 3. المستخدمون (Personas)

### P1: العميل (Customer)
يبحث عن الخدمات، يقدم الطلبات، يتتبع حالتها، يدفع إلكترونياً، يتواصل مع الدعم.

### P2: الموظف (Employee)
يدير الطلبات الواردة، يتواصل مع العملاء، يقدم الدعم الفني، يدير المحتوى.

### P3: المدير (Manager/Admin)
يشرف على العمليات، يتابع الأداء، يدير الموظفين والصلاحيات.

### P4: الشريك (Partner)
يتكامل عبر API، يقدم خدمات، يتابع الأداء.

### P5: المطور (Developer)
يبني تكاملات، يستخدم SDK و API Documentation.

## 4. رحلة المستخدم (User Journeys)

### رحلة العميل — طلب خدمة
1. يزور الموقع الإلكتروني
2. يتصفح الخدمات المتاحة
3. يختار الخدمة المناسبة
4. يملأ نموذج الطلب
5. يرفع المستندات المطلوبة
6. يدفع رسوم الخدمة
7. يستلم تأكيد الطلب
8. يتتبع حالة الطلب عبر لوحة التحكم
9. يستلم الخدمة ويقيمها

### رحلة الموظف — معالجة طلب
1. يستلم إشعار بطلب جديد
2. يراجع تفاصيل الطلب
3. يعالج الطلب ويحدث حالته
4. يتواصل مع العميل عند الحاجة
5. يغلق الطلب

## 6. حالات الاستخدام الأساسية (Core Use Cases)

### UC-01: تسجيل عميل جديد
الممثل: زائر. يدخل بياناته، يؤكد حسابه عبر البريد، يكمل ملفه الشخصي.

### UC-02: تقديم طلب خدمة
الممثل: عميل. يتصفح الخدمات، يختار الخدمة، يملأ النموذج، يدفع، يستلم تأكيداً.

### UC-03: إدارة طلب (موظف)
الممثل: موظف. يستلم الطلب، يراجع المستندات، يعالج، يحدث الحالة، يغلق.

### UC-04: شراء منتج
الممثل: عميل. يتصفح المتجر، يضيف للسلة، يكمل الشراء، يدفع، يتابع الشحن.

### UC-05: فتح تذكرة دعم
الممثل: عميل. يفتح تذكرة، يصف المشكلة، يتلقى رداً، تغلق التذكرة.

### UC-06: إنشاء تقرير
الممثل: مدير. يختار نوع التقرير، يحدد الفترة، يستعرض، يصدر.
