# سير العمل (Workflows) — منصة الحارس تك

---

## 1. سير عمل تطوير ميزة جديدة

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   الرؤية     │ →  │  المواصفات   │ →  │   التخطيط    │
│  (Vision)    │    │  (Spec)      │    │  (Plan)     │
└─────────────┘    └─────────────┘    └─────────────┘
                                               │
                                               ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   المراجعة   │ ←  │  الاختبار    │ ←  │   التطوير    │
│  (Review)    │    │  (Test)      │    │  (Develop)   │
└─────────────┘    └─────────────┘    └─────────────┘
        │
        ▼
┌─────────────┐    ┌─────────────┐
│    الدمج     │ →  │   النشر      │
│   (Merge)    │    │  (Deploy)    │
└─────────────┘    └─────────────┘
```

---

## 2. سير عمل تقديم طلب خدمة (Customer)

```
1. العميل يتصفح الخدمات
2. يختار الخدمة
3. يملأ نموذج الطلب + يرفع المستندات
4. يدفع (إن وجدت رسوم)
5. يستلم تأكيد ← إشعار بريد
6. الموظف يستلم الطلب ← إشعار
7. الموظف يراجع الطلب
   ├── مكتمل ← يعالج
   └── ناقص ← يطلب استكمال من العميل
8. يعالج الطلب
9. يحدث الحالة ← العميل يستلم إشعار
10. يكمل ← يقفل الطلب
11. العميل يقيم الخدمة
```

---

## 3. سير عمل تذكرة الدعم الفني

```
1. العميل يفتح تذكرة (يصف المشكلة + تصنيف)
2. النظام يعين أولوية تلقائياً
3. مدير الدعم يعين موظف
4. الموظف يستلم ← إشعار
5. الموظف يراجع ويبحث في قاعدة المعرفة
   ├── محلولة ← يرسل الحل
   └── تحتاج تدخل ← يعالج
6. العميل يؤكد الحل أو يطلب المزيد
7. تغلق التذكرة
8. تقييم الخدمة
```

---

## 4. سير عمل الشراء من المتجر

```
1. العميل يتصفح المنتجات
2. يضيف للسلة
3. يتجه للدفع
4. يدخل عنوان الشحن
5. يختار طريقة الدفع
6. يدفع ← تأكيد الدفع
7. النظام ينشئ طلب شراء
8. إشعار للعميل + إشعار للإدارة
9. تجهيز الطلب (المخزون)
10. شحن الطلب
11. العميل يتتبع الشحنة
12. استلام ← تقييم
```

## 5. سير عمل CI/CD

```
Push to feature branch
  │
  ▼
Lint + Type Check
  │
  ▼
Unit Tests
  │
  ▼
Integration Tests
  │
  ▼
Build
  │
  ▼
Open PR → Develop
  │
  ▼
Review + CI Checks
  │
  ▼
Merge to Develop
  │
  ▼
Deploy to Staging
  │
  ▼
E2E Tests on Staging
  │
  ▼
Merge to Main
  │
  ▼
Deploy to Production
```

## 6. سير عمل إدارة المحتوى

```
1. محرر المحتوى ينشئ/يعدل صفحة/مقال
2. معاينة المحتوى
3. حفظ كمسودة أو نشر
   ├── نشر فوري
   └── جدولة للنشر لاحقاً
4. إعادة بناء الصفحات الثابتة (ISR/SSG)
5. المحتوى يظهر على الموقع
```

## 7. سير عمل النسخ الاحتياطي

```
يومياً (تلقائي):
  1. نسخ احتياطي لقاعدة البيانات ← S3
  2. نسخ احتياطي للملفات ← S3
  3. الاحتفاظ بآخر 7 نسخ يومية
  4. الاحتفاظ بآخر 4 نسخ أسبوعية
  5. الاحتفاظ بآخر 3 نسخ شهرية
  6. إشعار عند فشل النسخ الاحتياطي
```
